# Caso de Uso – RF-03: Autenticação do usuário

## Descrição
O sistema deve permitir login.

## Atores
- Usuário

## Pré-condições
- Usuário ja cadastrado.
- Dados válidos.

## Fluxo principal
1. O ator acessa a tela de login.
2. O sistema exibe o que precisa ser preenchido.
3. O ator preenche os dados necessários (email e senha).
4. O sistema valida os dados.
5. O sistema confirma a operação.

## Fluxos alternativos
- Dados inválidos: o sistema exibe mensagem de erro.
- Operação cancelada: o sistema retorna à tela anterior.

## Pós-condições
- Usuário cadastrado.
