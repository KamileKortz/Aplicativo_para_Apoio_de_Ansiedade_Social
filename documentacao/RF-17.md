# Caso de Uso – RF-17: Controle de permissões

## Descrição
Diferenciar permissões entre responsável e criança.

## Atores
- Sistema.

## Pré-condições
- Tentativa de acesso a menu administrativo.

## Fluxo principal
1. O usuário solicita acesso a uma função (ex: exclusão de dados).
2. O sistema verifica se o perfil logado é de Responsável ou Criança.
3. Se for Criança, o sistema nega o acesso e pede a senha do responsável.
4. Se for Responsável, o sistema libera o acesso.

## Pós-condições
- Acesso permitido ou negado conforme o perfil.
