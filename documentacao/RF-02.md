# Caso de Uso – RF-02: Cadastro de usuário criança

## Descrição
O sistema deve permitir o cadastro de criança atrelado a uma conta de responsável.

## Atores
- Responsável

## Pré-condições
- Dados vãlidos.
- Responsável logado.

## Fluxo principal
1. O ator seleciona a opção "Cadastrar criança".
2. O sistema exibe o que precisa ser preenchido.
3. O ator preenche os dados necessários.
4. O sistema valida as informações e vincula com a conta do responsável.
5. O sistema confirma a operação.

## Fluxos alternativos
- Dados inválidos: o sistema exibe mensagem de erro.
- Operação cancelada: o sistema retorna à tela anterior.

## Pós-condições
- Dados salvos.
