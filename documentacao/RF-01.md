# Caso de Uso – RF-01: Cadastro de usuário responsável

## Descrição
O sistema deve permitir o cadastro de pais ou responsáveis, usando nome completo, email e senha.

## Atores
- Responsável

## Pré-condições
- Dados vãlidos.

## Fluxo principal
1. O ator seleciona a opção "Cadastrar-se".
2. O sistema exibe o que precisa ser preenchido.
3. O ator preenche os dados necessários (nome, email e senha).
4. O sistema valida os dados.
5. O sistema confirma a operação.

## Fluxos alternativos
- Dados inválidos: o sistema exibe mensagem de erro.
- Operação cancelada: o sistema retorna à tela anterior.

## Pós-condições
- Dados salvos.
